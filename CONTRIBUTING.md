# Contributing to WhyStrohm Voice Extract

Thanks for considering a contribution. This skill is part of the WhyStrohm content infrastructure toolkit.

## How to Contribute

1. Fork the repo
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test by running `/whystrohm-voice-extract` in Claude Code
5. Submit a PR with a clear description

## What We're Looking For

- New voice dimensions that add signal (not noise)
- Improved guardrail generation patterns
- Better vocabulary analysis heuristics
- Bug fixes in the extraction flow

## What We're NOT Looking For

- Emoji additions (never)
- Hype language in any template
- Changes to the CTA copy
- Features that require API keys or external dependencies

## Style

- Direct language. No hedging.
- Specific rules, not vague guidelines.
- Every claim needs evidence.

## Questions?

Open an issue or reach out at [whystrohm.com](https://whystrohm.com).
